# Day 13 - Debugging

This folder contains notes and learnings from **Day 13** of the **100 Days of Python Bootcamp**.

## Overview

Day 13 focused entirely on **debugging**—the process of identifying, understanding, and fixing errors in code.

Unlike previous days, no standalone project was built. Instead, the day was dedicated to improving problem-solving skills by working through buggy code examples and learning effective debugging techniques.

## Topics Covered

- Reading Python error messages
- Understanding tracebacks
- Finding syntax errors
- Finding logical errors
- Debugging loops and conditionals
- Using print statements for debugging
- Systematically testing code
- Developing a debugging mindset

## Key Takeaways

During this lesson, I learned that:

- Bugs are a normal part of programming.
- Error messages often provide valuable clues.
- Understanding the root cause is more important than applying quick fixes.
- Testing small pieces of code makes debugging easier.
- Debugging is a skill that improves with practice.

## Personal Reflection

This day helped me become more confident when approaching errors in my code. Rather than guessing solutions, I learned to investigate problems methodically and use debugging techniques to find the actual cause.
